# Домашнее задание к занятию "`Репликация и масштабирование. Часть 1`" - `Букавело Алексей`


### Задание 1
Опишите основные преимущества использования масштабирования методами:
активный master-сервер и пассивный репликационный slave-сервер;
master-сервер и несколько slave-серверов;

1.Активный master и пассивный slave:
Поскольку чтение и запись происходят на одном узле, исключена ситуация, когда пользователь записал данные и не увидел их при обновлении страницы. Не нужно настраивать логику распределения запросов.

2.master-сервер и несколько slave-серверов:
Огромная производительность на чтение - если ваш сайт читают 10 000 человек легко обрабатывать весь входящий трафик. Выход из строя одного или даже двух Slave-серверов пройдет незаметно для пользователей.Можно динамически добавлять или удалять Slave-серверы в зависимости от текущей нагрузки


### Задание 2
Разработайте план для выполнения горизонтального и вертикального шаринга базы данных. База данных состоит из трёх таблиц:

пользователи,
книги,
магазины (столбцы произвольно).
Опишите принципы построения системы и их разграничение или разбивку между базами данных.

Вертикальный шардинг:
Каждая таблица живёт в своей собственной базе данных на своём сервере, потому что они выполняют разные функции. Каждый из трех серверов работает в режиме Master для записи своих данных и для каждого из них можно добавить по Slave-серверу, чтобы распределять нагрузку чтения

Горизонтальный шардинг:
Горизонтальный шардинг по users_id, горизонтальный шардинг по location_ru/uk. Каждый shard работает в режиме Master для своего части таблицы

Так как каждая db лежит на отдельном сервере, объеденение через sql невозможно. Это разные физические БД, объединение происходит в приложении:
1.Запросы в shard ru location, shard uk location
2.Склейка в коде
