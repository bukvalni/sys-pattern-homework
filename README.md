### Задание 2

SYN-сканирование:

![SYN-сканирование](/img/SYN-сканирование.jpg)

1.Отправляется TCP пакет с флагом SYN
2.Полное трёхэтапное рукопожатие не завершается
3.Самый быстрый и часто используемый

При отправке SYN-пакета на открытый порт, сервер отвечает SYN/ACK, показывая, что готов установить соединение. Если порт закрыт, сервер сразу сбрасывает соединение, отправляя RST. Если порт фильтруется файрволом, ответа не будет вообще.



FIN-сканирование:

![FIN-сканирование](/img/FIN-сканирование.jpg)

1.Отправляется TCP пакет только с флагом FIN
2.Использует RFC-поведение: закрытые порты должны отвечать RST
3.Может обходить простые файрволы

При отправке пакета с флагом FIN. По стандартам TCP, если порт закрыт, сервер обязан ответить RST. Если порт открыт, сервер просто игнорирует этот пакет и ничего не отвечает. Это позволяет определить открытые порты по отсутствию ответа.

Xmas-сканирование:

![Xmas-сканирование](/img/Xmas-сканирование.jpg)

1.Отправляется TCP пакет с флагами FIN, PSH, URG
3.Также обходит некоторые файрволы

Отправляются пакеты с флагами FIN, PSH и URG одновременно. Логика та же: закрытые порты отвечают RST, открытые молчат.

UDP-сканирование:

![UDP-сканирование](/img/UDP-сканирование.jpg)

1.Отправляются UDP датаграммы
2.Определение основано на ICMP-ответах
3.Медленнее и менее надёжно из-за отсутствия ответа от открытых портов

Отправляется пустой UDP-пакет. Если порт закрыт, сервер возвращает ICMP-сообщение "Port Unreachable". Если порт открыт, чаще всего сервер ничего не отвечает, хотя некоторые службы могут отправить данные в ответ.
