### Задание 3

![Задание 3](img/ctrl+c.jpg)

Контейнер использует nginx в качестве главного процесса (PID 1). Комбинация Ctrl+C отправляет сигнал SIGINT этому процессу. Nginx завершает свою работу, и поскольку это был единственный главный процесс, контейнер тоже останавливается

![Задание 3](img/vim.jpg)

![Задание 3](img/curl%2081.jpg)

![Задание 3](img/check.jpg)

При создании контейнера мы пробросили порт 8080 хоста на 80 внутри контейнера (-p 8080:80). Когда мы поменяли listen 80 на 81, nginx перестал слушать 80-й порт, но проброс остался старым. Docker всё ещё направляет трафик с localhost:8080 на порт 80 контейнера, где никто не слушает, поэтому запрос зависает и возвращает ошибку
